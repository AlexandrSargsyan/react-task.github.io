Task

Create React Application

At main page user should see their meetings history. Data can be fetched from here: https://alexandrsargsyan.github.io/react-task.github.io/meetings.json

In this JSON keys are datetimes indicating meeting start time.Dates represented in YYYY-MM-DDTHH:mm:ss format

  • User should see all meetings in ascending order groupped by day
  • For every group user should see how many meetings in that day
  • Every group name should be formatted in following format Month name day, year example: Feb 25, 2022
  • Under each group every meeting for that day should be in ascending order
  • Every meeting has duration and name, user should be able to see start time, end time and name of the meeting
  • Duration represented in JSON with following format: amount+unit example: 15min, 1h, 1h 30min
  • User should be able to see past and upcoming meetings in respective groups

    • Page should have filter section where user should be able to filter meetings by following criterias:

    • Date range (start time and end time)
    • Upcoming meetings
    • Meetings in this week
    • Meetings which duration less than 30min
    • Online / Offline meetings

    When user clicks Filter button they should redirect to new page with filtered data

    Important! Filter information should be part of URL and if we change data in URL it should be affected on filter

    Reset button should reset all filters and show all available data

    Functional Requirements

    • User should see some information about loading before data will be available, if data not returned user should see appropriate message
    • If user opened page and there is meeting that starts in less than 30 minutes they should be informed
    • Data should be cached for 10 minutes

    Technical Requirements

    • App should be written in Typescript and with newer version of React
    • All Components should be reusable as much as possible
    • Presence of Unit Tests is a big plus
